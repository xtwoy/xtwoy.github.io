记一次 Docker Compose 内部容器通信失败问题

前言

最近，我对 Himit_ZH 的 Himit_ZH / HOJ 项目进行了二次开发。

在了解项目的过程中，我选择使用 Docker Compose 来容器化部署。Docker Compose 提供了一种简便的方式来定义、配置和启动多个容器，使我能够快速搭建起整个系统。

然而，在启动容器时，我遇到了一个令人困惑的问题：内部容器之间的通信失败。特别是在 HOJ 项目的 MySQL 数据库容器和 Nacos 注册中心容器之间的通信出现了问题。这导致 HOJ 无法正常运行，并且我花费了相当一段时间来诊断和解决这个问题。

经过仔细排查和调试，我最终发现问题出在容器之间的网络配置上。默认情况下，Docker Compose 创建了一个名为 hoj-network-1 的网络，其中包含了 HOJ 项目中的所有容器。然而，由于我在配置中错误地设置了 MySQL 容器的端口映射，导致 Nacos 容器无法通过指定的端口访问 MySQL 服务。

部署

尝试访问 linux:8848，linux拒绝访问。

排查

  1. 使用 docker logs hoj-nacos -n 100 查看hoj-nacos的运行日志。

    没有数据源的设置？ 已经设置，不存在，应该是连接不了。

  2. 在物理主机使用 SQLyog 连接 hoj-mysql ，没有问题。

  3. 使用 docker network inspect standalone_hoj-network-1 看它们是否连接到自定义网咯，没有问题。

解决

使用未修改的的 docker-compose.yml 。hoj-nacos 启动成功。

对比发现

MYSQL_SERVICE_HOST自定义网段不同而已，不影响。

MYSQL_SERVICE_PORT一个使用 默认端口 ，一个使用 暴露端口。

可是我记得我一开始并没有去改 MYSQL_SERVICE_PORT。也 docker-compose 一次。同样出现了这样的问题，所以才怀疑是我改了端口映射的问题，改成了将hoj-nacos访问hoj-mysql的端口改为3406。

网上找到了一篇帖子，原来 在docker网络中运行所有内容 (redis、replica和spring) 时，应该使用端口6379而不是映射端口
