实现跨机器访问：在局域网下固定虚拟机 IP，访问运行在 VMware 虚拟机中的服务

导语：在同一个局域网中，通过设置和配置，让机器 A 上的服务可以被机器 B 访问是一项重要的任务。本文将提供简单明了的指南，帮助你实现这一目标。

正文：

在局域网中，将机器 A（运行有 VMware 和 Linux 虚拟机）和机器 B 连接到同一个网络，从而实现 B 访问 A 上的虚拟机上的服务非常重要。以下是一些简单的配置步骤，帮助你完成这个设置：

  1. 连接机器 A 和机器 B 到同一个局域网： 确保机器 A 和机器 B 都连接到同一个局域网。可以通过有线或无线连接将它们连接到同一个网络交换机或路由器上。
  2. 配置机器 A 上的 VMware 网络设置： 在机器 A 上，打开 VMware 软件并进入虚拟机网络设置。选择桥接模式，并选择正确的物理网络接口，以便虚拟机可以直接连接到局域网中的其他设备。
  3. 在虚拟机中，确保虚拟机的网络设置正确。为虚拟机分配一个固定的 IP 地址，该 IP 地址应与机器 A 在局域网中的 IP 地址处于同一个子网中。您可以在虚拟机的操作系统中配置静态 IP 地址，确保与局域网中的其他设备保持一致。并确保子网掩码、默认网关和 DNS 设置正确。
  4. 启动服务并监听正确的 IP 地址： 在虚拟机中，确保你的服务正在运行，并且监听的 IP 地址是机器 A 在局域网中的 IP 地址。
  5. 在物理机 B 上访问虚拟机 A 上的服务： 在物理机 B 上，使用虚拟机 A 的 IP 地址和服务的端口号，在浏览器或其他客户端工具中访问虚拟机 A 上的服务。例如，如果虚拟机 A 的 IP 地址为 192.168.3.XX，端口号为 8080，则在物理机 B 的浏览器中访问 http://192.168.3.XX:8080。

1、2、3、5操作很简单，不做介绍了。

    假设您的笔记本 IP 地址为 192.168.3.20，子网掩码为 255.255.255.0。

几个问题

为什么要在同一个网段

  1. 如果将 VMware 设置为桥接模式，并且虚拟机的 IP 地址与物理机不在同一个网段，可能会导致网络连接问题，使得虚拟机与物理机之间无法直接通信。

    当您将 VMware 设置为桥接模式时，虚拟机的网络适配器将与物理网络中的其他设备直接连接，就好像它是一个独立的物理设备。这意味着虚拟机需要在物理网络中具有唯一的 IP 地址，并与其他设备在同一个网段中。

    如果虚拟机的 IP 地址与物理机不在同一个网段，可能会发生以下情况：

    1. 虚拟机无法访问物理网络或互联网：如果虚拟机的 IP 地址不在与物理机相同的网段中，它可能无法正确路由到物理网络或互联网上的其他设备。这将导致虚拟机无法访问外部网络服务或进行网络通信。

    2. 物理机无法访问虚拟机：同样地，如果虚拟机的 IP 地址与物理机不在同一个网段中，物理机可能无法直接访问虚拟机中运行的服务或应用程序。

    要解决这个问题，您可以考虑以下选项：

    1. 将虚拟机的 IP 地址设置为与物理机在同一个网段中，确保它们能够直接通信。
    2. 如果有网络设备（如路由器）充当虚拟机和物理机之间的网关，您可以配置正确的路由表和网络转发规则，以使虚拟机和物理机能够互相访问。
    3. 如果无法在同一个网段中设置虚拟机的 IP 地址，您可以考虑使用其他网络模式，如 NAT 模式或仅主机模式。这些模式将允许虚拟机与物理机进行通信，而无需与物理网络中的其他设备直接连接。

VMware 中的桥接模式和 NAT 模式是两种不同的网络连接模式，它们具有以下区别和使用场景：

  1. 桥接模式（Bridge Mode）：

    • 桥接模式将虚拟机的网络适配器连接到主机物理网络适配器上，使虚拟机直接与物理网络上的其他设备通信。
    • 虚拟机在桥接模式下获得的 IP 地址是与物理网络中的设备处于同一子网的地址。
    • 虚拟机在桥接模式下可以直接与局域网中的其他设备进行通信，包括访问局域网中的服务和共享文件。
    • 桥接模式提供了更直接和透明的网络连接，适用于需要虚拟机直接参与局域网中的网络活动的情况。

  2. NAT 模式（Network Address Translation Mode）：

    • NAT 模式通过创建一个虚拟的网络子网，将虚拟机的网络流量转发到主机上的 NAT 网络。
    • 虚拟机在 NAT 模式下获得的 IP 地址是由 VMware 动态分配的私有 IP 地址。
    • NAT 模式隐藏了虚拟机背后的真实 IP 地址，使用主机的 IP 地址来与外部网络通信。
    • 虚拟机在 NAT 模式下可以通过主机的网络连接访问外部网络和互联网，但无法直接与局域网中的其他设备进行通信。
    • NAT 模式提供了一定程度的网络隔离和安全性，适用于需要虚拟机与外部网络隔离并使用主机作为代理进行网络连接的情况。

为什么大部分用户选择 NAT 模式而不是桥接模式呢？这是因为 NAT 模式相对来说更为简单和安全，尤其适用于个人用户或测试环境中的虚拟机。以下是一些常见原因：

  1. 方便的互联网访问：NAT 模式使虚拟机能够通过主机的网络连接直接访问互联网，无需复杂的网络配置，方便了虚拟机的互联网访问。

  2. 简化网络设置：NAT 模式隐藏了虚拟机的真实 IP 地址，并使用主机的 IP 地址与外部网络通信。这简化了网络设置和防止冲突的需要。

  3. 安全性：NAT 模式提供了一定程度的网络隔离，虚拟机的真实 IP 地址对外部网络是不可见的，减少了虚拟机直接暴露在物理网络中的风
