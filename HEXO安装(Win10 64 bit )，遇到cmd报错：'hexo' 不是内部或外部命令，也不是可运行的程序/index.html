HEXO安装(Win10 64 bit )，遇到cmd报错：‘hexo’ 不是内部或外部命令，也不是可运行的程序

由于使用要使用多个版本的node，下载了个nvm。切换了系统使用的node版本。过一段时间，使用hexo更新博客的时候，发出**'hexo’不是内部或外部命令，也不是可运行的程序**的报错。node换回原来的版本也是不行。甚至以前下载的一系列工具都使用不了（PS：vue)。

折腾来折腾去一直不行，索性重开。😡😡😡😡

  1. 使用nvm下载 v16.16.0的node

  2. 使用v16.16.0的node，配置淘宝镜像。啥node_cache node_global根本不用设置 方便的很

  3. 安装hexo啦

    安装的2个命令

    npm install hexo-cli -g

    npm install hexo --save

  4. 查看版本的命令

    hexo -v

    然后寄了

看别人的例子呢，到这都是一帆风顺，程序正常执行，但是到我这就，谢谢你node！！！！

下面是解决之道，经过我一番搜索，实验。然后找到了解决方法。

说是要配置下环境变量。

下面，咱先仔细看下，安装完成之后的，目录。

之前看到好几个文章说吧hexo的bin目录添加到环境变量。但是因为啥原因hexo.cmd并没有放到bin目录里。导致我这一直不行。

哦，这么一看，大概就只怎么配置环境变量了吧。

这个之后，咱再运行一次，hexo命令。

OK，可算是成功了。

哈哈哈哈，我Windows可算是可以运行这玩意儿啦。

嗯，可以继续吐槽node，两天花六小时弄node。

总结出现xxx不是内部或外部命令，也不是可运行的程序，就把有xxx.cm的目录路径添加到环境变量去。
